Hosting & Auth Platform Comparison

Static and dynamic website hosting with email/password + SSO authentication across GCP, Azure, Cloudflare, and Wix. Static is sized by total members (auth MAU); dynamic is sized by concurrent users (people active at the same instant), scaling from 10 to 2,000 as the community grows.

📄 Static hosting serves pre-built HTML/CSS/JS files from a CDN. No server runs per request, so a CDN absorbs thousands of concurrent visitors at no extra cost — concurrency is essentially a non-issue here, and cost is driven by total members (auth MAU), not simultaneous load. Auth and database are handled by separate third-party services (Supabase, Firebase, Entra). Lower cost, faster delivery, easier to maintain. Best fit for most community websites at this scale.

⚡ Dynamic hosting runs a server that renders pages per request — real-time data, server-side logic, personalized responses, API routes, with auth handled server-side. Costs below assume an always-on 24/7 server sized for sustained concurrent load, because that is the predictable way to serve a large simultaneous audience without cold-start lag. Note: "concurrent users" (active at the same instant) is far more demanding than total members — at 2,000 concurrent the free serverless tiers are nowhere near enough, so dynamic costs grow with provisioned capacity.
📐 Sizing assumptions (estimates — verify with each provider's calculator). Rule of thumb: ~1 vCPU per 10–20 active in-flight requests. 10 concurrent → one small always-on instance (1–2 vCPU). 500 concurrent → 4–8 vCPU, 1–2 redundant instances + managed DB. 2,000 concurrent → horizontally scaled 16–32 vCPU across multiple instances + load balancer + high-availability database. Real capacity depends on your code; treat all figures as order-of-magnitude monthly estimates before nonprofit credits.
